# LIFE OS Morning / Task 再設計 実装指示書

## 1. 目的

今回の改修の目的は、`LIFE OS` をより「朝の起動」と「日中の実務整理」に特化させ、スマホでの情報ノイズと入力負荷をさらに減らすことにある。

特に以下を同時に達成する。

1. `Morning` を「朝ルーティンをこなして1日を起動する画面」に再定義する
2. 昨日からの引き継ぎ情報は `Task` に集約し、`Morning` から外す
3. `Task` 画面で、今日やることと持ち越しを一目で整理できるようにする
4. ボタン・通知・未整理状態の視認性を強化し、押したかどうか・未完了がどこかを直感でわかるようにする
5. 情報ノイズを消し、「1カード1タイトル」の原則をこの OS の絶対ルールとして固定する

## 3. 全体デザイン原則

## 3.1 絶対ルール

この OS の UI において、**同じカードの中に英語見出しと日本語見出しを二重で並べない**。

原則:

- 1カードにつきタイトルは1つだけ
- キッカー英語 + 本タイトル日本語の二重見出しは禁止
- 情報価値のない飾り見出しは削除する
- 画面を開いた瞬間に、次にやることがわかることを最優先する

適用対象:

- `Morning Focus`
- `Carry Over`
- `Contact`
- `Task`
- `121`
- その他すべてのカード

推奨:

- タイトルは日本語に統一
- 必要なら小さな補足は本文や注記で表現する

## 4. Morning 画面の再設計

## 4.1 Handoff を Morning から削除

`Morning` から `Hand off / 昨日からの引き継ぎ` を完全に削除する。

理由:

- 朝に最初に見る情報としてはノイズが大きい
- Morning の責務は「ルーティンをこなし、今日を起動すること」
- 引き継ぎ確認は実務に入る `Task` 画面で見る方が自然

変更内容:

- `Morning` 画面の Handoff カードを削除
- それに伴い `renderHandoff()` の Morning 側描画は削除
- Handoff 内容は Task 側の統合カードへ移設する

## 4.2 Morning の優先順位を固定

`Morning` を開いた瞬間に最優先で目に入るのは朝ルーティンのみとする。

画面順序は以下に変更する。

1. 日付 / 進捗
2. 朝ルーティン
3. 今週の目標
4. 今日の最重要3つ
5. `1日をはじめる` ボタン

ルール:

- 朝ルーティン完了前に、引き継ぎや雑多な情報を見せない
- ルーティンは最上段
- `今週の目標` と `最重要3つ` はルーティンの後

## 4.3 朝ルーティンの自動折りたたみ

朝ルーティン項目は、チェック完了したらその場で折りたたまれる仕様に変更する。

目的:

- スクロール量の削減
- 完了済み情報を視界から外し、未完了だけに集中させる

仕様:

- 未完了項目は通常表示
- 完了した項目はアニメーション付きで圧縮表示に切り替える
- 圧縮後は高さを小さくし、1行要約だけ残してよい
- 全件完了時は `朝ルーティン完了` の要約帯にまとめてもよい

実装ルール:

- `renderRoutineList()` に `collapseCompleted` オプションを追加して朝だけ有効化する
- `night` にはこの挙動を適用しない

## 4.4 最重要1への自動反映

`未来からの今日の一手` は `Morning` の `今日の最重要3つ` の `最重要1` に自動反映する仕様へ変更する。

変更内容:

- `最重要1に反映する` ボタンは削除
- `Time Machine` 完了時、または `Morning` 描画時に `mit1` へ自動反映する
- `mit2` と `mit3` はユーザーが当日入力する

反映ルール:

1. `futureVision.todayActionFromFuture` に値がある
2. `mit1` はその値で同期する
3. ユーザーは `mit2` と `mit3` のみ自由入力する

運用前提:

- `mit1` を Morning 上で編集不可にするか、編集可でも「未来からの今日の一手」に常に同期するかを実装前に決める必要がある

今回の推奨:

- `mit1` は readonly に近い扱いにし、Morning 上では表示専用または薄いロック表示にする
- 理由:
  - 朝の軸をぶらさない
  - ユーザー判断は `mit2` `mit3` に寄せる

## 4.5 ボタン文言変更

`タスクへ進む` は `1日をはじめる` に変更する。

理由:

- 朝の画面の役割はタスク入力ではなく、1日の起動
- ユーザー体験として自然

内部動作:

- 遷移先は引き続き `Task` 画面でよい
- ボタンラベルのみ変更

## 5. Task 画面の再設計

## 5.1 CARRY OVER と HAND OFF の統合

`CARRY OVER` と `HAND OFF` は統合し、**昨日決めた今日やること一覧**として表示する。

表示内容:

- 前日 `nextAction`
- 前日の `tomorrowReminder`
- `carryOver` 状態で持ち越された Day Ops 項目

目的:

- 「昨日の自分が今日に渡したもの」を1箇所で見る
- 実務に入る前の確認画面を Task に集約する

表示仕様:

- 1枚の統合カードにする
- セクションタイトルは日本語1つだけにする
- 例:
  - `昨日からの引き継ぎ`

カード内の表示順:

1. 今日のアクション
2. 昨日のリマインド
3. 持ち越しタスク一覧

持ち越し機能:

- `carryOver` 項目だけを数える既存機能は維持
- 持ち越しがない場合は `持ち越しなし` を表示

## 5.2 セクション名変更

`121日程調整` はすべて `日程調整` に変更する。

対象:

- 画面タイトル
- セクション名
- 通知テキスト
- Markdown 出力内の見出し
- `DAY_OPS_TYPE_CONFIG.label`

内部キー:

- `schedule121` はそのままでよい

## 5.3 折りたたみ時の件数通知を強化

折りたたまれた `連絡 / 日程調整 / 作業タスク` に1件以上項目がある場合、件数バッジをもっと目立たせる。

見た目仕様:

- 薄緑ベース
- ほんのり発光
- 未読通知のように視線を引く
- 暗い背景でも目立つ

対象:

- 総件数
- 未整理件数

優先度:

1. 未整理件数がある場合は最優先で強調
2. 未整理はなくても件数が1件以上あるなら弱めに強調

推奨表現:

- `3件`
- `未整理1`

スタイル指示:

- `box-shadow` で外側グロー
- `background` は緑系半透明
- テキストは明るい緑または白

## 5.4 作業タスクに編集機能を追加

`作業タスク` に `編集` を追加する。

対象:

- 少なくとも `task` type に対して編集可能

編集項目:

- 作業
- ゴール
- 何分かかるか
- いつやるか

UI 方針:

- `編集` ボタンを押したら、そのカードだけインライン編集フォームに変わる
- もしくは、既存の追加フォームに値を流し込んで編集モードにする

今回の推奨:

- 既存追加フォームを再利用する編集モードを採用

理由:

- DOM を増やしすぎずに済む
- モバイルで扱いやすい
- 実装の重複が少ない

必要な state:

- `editingDayOpsItemId`
- `editingDayOpsType`

ボタン:

- `編集`
- `保存`
- `キャンセル`

## 5.5 1カード1タイトルの強制適用

Task 画面では、各カード冒頭の英語見出しを削除し、日本語タイトルだけにする。

削除対象の例:

- `Morning Focus`
- `Carry Over`
- `Contact`
- `Task`
- `121`
- `Expense`

残すもの:

- `今日の最重要3つ`
- `昨日からの引き継ぎ`
- `連絡`
- `日程調整`
- `作業タスク`
- `支出メモ`

## 5.6 ボタンの状態を全面色分けに変更

現状はボタンが押せるか押せたかが弱く、視認性が低い。

変更内容:

- 完了系ボタンはボタン全体を色付きにする
- 他のボタンも役割ごとに全面色分けする
- 少し発光する近未来系 UI に寄せる

推奨ルール:

- `完了`: 緑系グロー
- `明日に回す`: 青系グロー
- `戻す`: 金系または黄系
- `削除`: 赤系弱グロー
- `編集`: 紫は避け、シアン寄りの寒色

状態:

- 通常
- hover
- active
- disabled
- selected / 実行済み

重要:

- 「押したあと色が残る」設計を取り入れる
- とくに `完了` はカード全体またはボタン全体に成功状態が残ると達成感が出る

## 5.7 未整理通知の赤い発光

`夜ルーティンへ進む` をブロックする未整理項目がある場合、その `未整理` 表示をほんのり赤く光らせる。

対象:

- セクションヘッダーの `未整理1`
- 各カードの `未整理` ステータスバッジ
- 画面下部の `未整理 n 件`

意図:

- 進めない理由を一瞬でわかるようにする
- ユーザーがどこを直せばよいか迷わないようにする

見た目:

- 薄い赤
- 弱いグロー
- 危険すぎないが、未完了感は明確

## 8. 受け入れ条件

以下をすべて満たしたとき、この改修は完了とみなす。

1. `Morning` に `昨日からの引き継ぎ` が存在しない
2. `Morning` を開くと朝ルーティンが最優先で見える
3. 朝ルーティン完了項目が折りたたまれる
4. `未来からの今日の一手` が `最重要1` に自動反映される
5. `タスクへ進む` が `1日をはじめる` に変わっている
6. `Task` 画面で `今日のアクション / 昨日のリマインド / 持ち越し項目` を1箇所で見られる
7. `121日程調整` がすべて `日程調整` に変わっている
8. `作業タスク` に `編集` がある
9. Task 画面の各カードに英語 + 日本語の二重タイトルが残っていない
10. 折りたたみセクションの件数バッジが薄緑発光で目立つ
11. `完了 / 明日に回す / 戻す / 削除 / 編集` のボタンが全面色付きで判別しやすい
12. 未整理項目があると、その表示がほんのり赤く光る
13. `夜ルーティンへ進む` のブロック仕様は維持され、未整理箇所も視覚的にわかる
